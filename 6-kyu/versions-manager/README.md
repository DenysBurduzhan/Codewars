<h2><a href=https://www.codewars.com/kata/5bc7bb444be9774f100000c3/train/java target="_blank">Versions manager</a></h2><h3>6 kyu</h3><p>In this kata we are going to mimic a software versioning system.</p><p>You have to implement a <code>VersionManager</code> class.</p><p>It should accept an optional parameter that represents the initial version. The input will be in one of the following formats: <code>"{MAJOR}"</code>, <code>"{MAJOR}.{MINOR}"</code>, or <code>"{MAJOR}.{MINOR}.{PATCH}"</code>. More values may be provided after <code>PATCH</code> but they should be ignored. If these 3 parts are not decimal values, an exception with the message <code>"Error occured while parsing version!"</code> should be thrown. If the initial version is not provided or is an empty string, use <code>"0.0.1"</code> by default.</p><p>This class should support the following methods, all of which should be chainable (except <code>release</code>):</p><ul><li><code>major()</code> - increase <code>MAJOR</code> by <code>1</code>, set <code>MINOR</code> and <code>PATCH</code> to <code>0</code></li><li><code>minor()</code> - increase <code>MINOR</code> by <code>1</code>, set <code>PATCH</code> to <code>0</code></li><li><code>patch()</code> - increase <code>PATCH</code> by <code>1</code></li><li><code>rollback()</code> - return the <code>MAJOR</code>, <code>MINOR</code>, and <code>PATCH</code> to their values before the previous <code>major</code>/<code>minor</code>/<code>patch</code> call, or throw an exception with the message <code>"Cannot rollback!"</code> if there's no version to roll back to. Multiple calls to <code>rollback()</code> should be possible and restore the version history</li><li><code>release()</code> - return a string in the format <code>"{MAJOR}.{MINOR}.{PATCH}"</code></li></ul><p>May the binary force be with you!</p>
