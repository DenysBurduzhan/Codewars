This is part 3 in a series of 4.</p><ul><li>Part 1: <a href="http://www.codewars.com/kata/java-functional-programming-part-1-the-beginning/java" data-turbolinks="false" target="_blank">http://www.codewars.com/kata/java-functional-programming-part-1-the-beginning/java</a></li><li>Part 2: <a href="http://www.codewars.com/kata/java-functional-programming-part-2-multiline-functions" data-turbolinks="false" target="_blank">http://www.codewars.com/kata/java-functional-programming-part-2-multiline-functions</a></li><li>Part 4: <a href="http://www.codewars.com/kata/java-functional-programming-part-4-row-row-row-your-boat-gently-down-the-dot-dot-dot" data-turbolinks="false" target="_blank">http://www.codewars.com/kata/java-functional-programming-part-4-row-row-row-your-boat-gently-down-the-dot-dot-dot</a></li></ul><p>We've now got a handle on Java lambda functions (even multi-line ones), but do Java lambdas have all the features of scripting functions? Like closures?</p><p>Yes. And, like in every language that supports them, closures give people headaches. Let's learn about closures!</p><h2 id="what-is-a-closure">What is a closure?</h2><p>In programming, a closure is a function that retains the context in which it was created.</p><p><em>What.</em></p><p>Okay, say we have these statements:</p><pre><code class="language-java"><span class="cm-type">int</span> <span class="cm-variable">foo</span> <span class="cm-operator">=</span> <span class="cm-number">5</span>;<span class="cm-variable">Function</span><span class="cm-operator">&lt;</span><span class="cm-type">String</span>, <span class="cm-type">Integer</span><span class="cm-operator">&gt;</span> <span class="cm-variable">f</span> <span class="cm-operator">=</span> <span class="cm-variable">s</span> <span class="cm-operator">-&gt;</span> <span class="cm-variable">s</span>.<span class="cm-variable">length</span> <span class="cm-operator">+</span> <span class="cm-variable">foo</span>;<span class="cm-variable">f</span>.<span class="cm-variable">apply</span>(<span class="cm-string">"somestring"</span>); <span class="cm-comment">//Should be 15</span></code></pre><p>Given an input of "somestring", f should return 15. But what happens when f leaves this context?</p><pre><code class="language-java"><span class="cm-keyword">public</span> <span class="cm-type">void</span> <span class="cm-def">someMethod</span>() {  <span class="cm-type">int</span> <span class="cm-variable">foo</span> <span class="cm-operator">=</span> <span class="cm-number">5</span>;  <span class="cm-variable">Function</span><span class="cm-operator">&lt;</span><span class="cm-type">String</span>, <span class="cm-type">Integer</span><span class="cm-operator">&gt;</span> <span class="cm-variable">f</span> <span class="cm-operator">=</span> <span class="cm-variable">s</span> <span class="cm-operator">-&gt;</span> <span class="cm-variable">s</span>.<span class="cm-variable">length</span> <span class="cm-operator">+</span> <span class="cm-variable">foo</span>;  <span class="cm-variable">f</span>.<span class="cm-variable">apply</span>(<span class="cm-string">"somestring"</span>); <span class="cm-comment">//Should be 15</span>  <span class="cm-variable">sendMethod</span>(<span class="cm-variable">f</span>);}<span class="cm-keyword">public</span> <span class="cm-type">void</span> <span class="cm-def">sendMethod</span>(<span class="cm-variable">Function</span><span class="cm-operator">&lt;</span><span class="cm-type">String</span>, <span class="cm-type">Integer</span><span class="cm-operator">&gt;</span> <span class="cm-variable">f</span>) {  <span class="cm-variable">f</span>.<span class="cm-variable">apply</span>(<span class="cm-string">"otherstring"</span>); <span class="cm-comment">//Should return 16</span>}</code></pre><p>Does f still know foo is 5 when f is in sendMethod? foo is a local variable to someMethod, after all. Yes, f knows 5 is 5; that's what closures do.</p><p>Write a factory that creates adder functions. Depending on what is passed to the create method, the function will, when called, add that amount to another amount given to the function. In other words:</p><pre><code class="language-java"><span class="cm-variable">f</span> <span class="cm-operator">=</span> <span class="cm-variable">AdderFactory</span>.<span class="cm-variable">create</span>(<span class="cm-number">1</span>); <span class="cm-comment">//Adds 1 to everything</span><span class="cm-variable">f</span>.<span class="cm-variable">applyAsInt</span>(<span class="cm-number">4</span>) <span class="cm-operator">==</span> <span class="cm-number">5</span>;<span class="cm-variable">f</span>.<span class="cm-variable">applyAsInt</span>(<span class="cm-number">10</span>) <span class="cm-operator">==</span> <span class="cm-number">11</span>;<span class="cm-variable">f</span>.<span class="cm-variable">applyAsInt</span>(<span class="cm-operator">-</span><span class="cm-number">5</span>) <span class="cm-operator">==</span> <span class="cm-operator">-</span><span class="cm-number">4</span>;</code></pre><p>A full listing of the default function types can be found at <a href="http://docs.oracle.com/javase/8/docs/api/java/util/function/package-summary.html" data-turbolinks="false" target="_blank">http://docs.oracle.com/javase/8/docs/api/java/util/function/package-summary.html</a></p><p>As in Part 2, don't use the default Function class, but the most appropriate class: the one that takes an int and returns an int; you shouldn't need to use any generics.</p>
